Welcome to the Vulnerable Web Application Demo

This application has been intentionally designed with several security vulnerabilities for educational purposes:

  • SQL Injection: Try the login form with malicious SQL input
  • Cross-Site Scripting (XSS): Post malicious JavaScript in the comments section
  • Insecure Direct Object References (IDOR): Access user profiles by manipulating the URL
  • Path Traversal: Upload files to unauthorized locations using the file upload feature
  • Command Injection: Execute arbitrary system commands in the Admin Tools section
  • Hidden Flag: Find and access the hidden flag file in the restricted directory
  • Security Misconfiguration: The application is running in debug mode

Exploitation Scenario

This application simulates several attack vectors:

  1. Use SQL injection to gain access to an account
  2. Upload a reverse shell script through the file upload vulnerability
  3. Use command injection to execute system commands or establish a reverse shell
  4. Find and access the flag hidden in the restricted directory

⚠️ Warning

This application is intentionally vulnerable and should only be used in a controlled environment for educational purposes.

DO NOT deploy this application in a production environment or use it with real user data.
